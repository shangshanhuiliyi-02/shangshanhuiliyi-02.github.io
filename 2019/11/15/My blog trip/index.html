My blog trip

2019-11-15
本文总阅读量次

1%~10%

一个小前端攻城狮,写着Html、Css、Js，老师开始教他node、ajax、vue实现前端与后台的交互 ，写本地blog。
他突发奇想，既然能写本地blog，为何不写一个个人blog呢 ？
说干就干 ！那么问题来 ！该怎么写个人blog呢？百度! 对， 百度！！
百度搜索 …“如何搭建个人博客” …嗯 。嗯 ！嗯 ？（挑选教程中）
“五分钟搭建个人博客”就决定是你了！

10%~20%

先是注册Github账号
—a few moments later—
输入邮箱、设置密码、验证邮箱，注册完成。
然后是创建仓库，仓库名必须和用户名比配。
第一步完成

20%~30%

接下来是安装工具 Node 、Git 、Hexo 。
wait ，Node 我知道 git、 hexo是什么？（黑人问号.jpg）

—a few moments later—（百度百科中）
恍然大悟 ，了解了 ，老老实实安装
测试是否安装成功
打开cmd 输入 node -v 、git -v 、hexo -v 、
嗯[摸下巴]好像有点不对 ，没有安装成功。
百度一下 [ 吓！] 为什么还要新建和更改环境变量 ？
行吧 [无奈] 还好是图文，按照步骤做吧。
[长叹一口气]终于解决了

30%~50%

然后设置本地用户信息 、 主题安装 、 修改配置文件。
照着葫芦画瓢 ， 也算是完成了。
然后是写文章 ， 新建一个.md文件（又卡住了）无法新建.md文件。
老规矩 —百度 —“windows10如何新建.md文件”
—a few moments later—
进入注册表编辑器–>新建项–>新建字符串–>OK
自由发挥 、天马行空 、随心所欲，写了第一个bolg（实则是按着教程copy的）
开始测试 $ hexo s 启动服务器 ，在浏览器输入https://localhost:4000 访问
成了 [暗松一口气 还以为会接着出错呢]
安装自动部署发布工具 （傻瓜式操作）

50%~70%

接下来就是发布了生成静态网页文件发布至github pages 中
$ hexo clean && hexo g && hexo d
上面说是第一次操作终端会让输入Github 的邮箱和密码，
可是为什么没让我输入而且还有一串红色代码 ，不出意外又出错了。
老办法–百度搜索–没有找到 ！这怎么办呢 ？对了 !还有b站!
对照着视频 ,一帧一帧的检查!
经历千辛万苦终于解决了终于把我的第一个博客发表出去了!

ps：每一段的百分比是来源于电影《超体》的灵感 至于为什么没有到100%是因为这只是开始好有很多要学习
